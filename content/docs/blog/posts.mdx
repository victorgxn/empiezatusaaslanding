# Crear Posts

Aprende a escribir artículos para tu blog usando MDX, que combina Markdown con componentes React.

## Estructura de un post

Cada post es un archivo `.mdx` en la carpeta `content/blog/`:

```mdx title="content/blog/mi-articulo.mdx"
---
title: Cómo lanzar tu SaaS en 7 días
description: Una guía paso a paso para ir de idea a producto lanzado
date: 2024-01-15
author:
  name: Tu Nombre
  avatar: /authors/tu-nombre.jpg
  twitter: "@tunombre"
category: tutorial
tags: [saas, lanzamiento, indie-hacking]
image: /blog/lanzar-saas.jpg
imageAlt: Desarrollador trabajando en su SaaS
published: true
---

# Cómo lanzar tu SaaS en 7 días

Tu contenido aquí...
```

## Frontmatter (metadatos)

El frontmatter define los metadatos del post. Todos los campos disponibles:

| Campo | Tipo | Requerido | Descripción |
|-------|------|-----------|-------------|
| `title` | string | ✅ | Título del artículo |
| `description` | string | ✅ | Descripción para SEO (150-160 caracteres) |
| `date` | string | ✅ | Fecha de publicación (YYYY-MM-DD) |
| `author` | object | ✅ | Información del autor |
| `author.name` | string | ✅ | Nombre del autor |
| `author.avatar` | string | ❌ | URL de la imagen del autor |
| `author.twitter` | string | ❌ | Handle de Twitter |
| `author.bio` | string | ❌ | Biografía corta |
| `category` | string | ✅ | Categoría principal |
| `tags` | string[] | ❌ | Lista de tags |
| `image` | string | ❌ | Imagen destacada |
| `imageAlt` | string | ❌ | Alt text de la imagen |
| `published` | boolean | ❌ | Si está publicado (default: true) |

### Ejemplo de frontmatter completo

```yaml
---
title: "Guía completa de Stripe para SaaS"
description: "Aprende a integrar pagos con Stripe en tu SaaS: checkout, suscripciones, webhooks y más"
date: 2024-02-20
author:
  name: María García
  avatar: /authors/maria.jpg
  twitter: "@mariadev"
  bio: "Desarrolladora full-stack y creadora de SaaS"
category: pagos

  - stripe

  - webhooks
```

## Sintaxis Markdown

### Headings

```markdown
# H1 - Solo para el título principal (generado automáticamente)
## H2 - Secciones principales
### H3 - Subsecciones
#### H4 - Detalles
```

```markdown

~~Tachado~~
```


## Componentes MDX

MDX permite usar componentes React dentro del Markdown. Crea componentes personalizados para tu blog:

### Componente Callout

```typescript title="components/blog/mdx/callout.tsx"
interface CalloutProps {
  type?: 'info' | 'warning' | 'error' | 'success';
  title?: string;
  children: React.ReactNode;
}

export function Callout({ type = 'info', title, children }: CalloutProps) {
  const styles = {
    info: 'bg-blue-500/10 border-blue-500 text-blue-500',
    warning: 'bg-yellow-500/10 border-yellow-500 text-yellow-500',
    error: 'bg-red-500/10 border-red-500 text-red-500',
    success: 'bg-green-500/10 border-green-500 text-green-500',
  };

  return (
    <div className={`p-4 rounded-lg border-l-4 my-6 ${styles[type]}`}>
      {title && <p className="font-semibold mb-2">{title}</p>}
      <div className="text-foreground">{children}</div>
    </div>
  );
}
```

### Componente YouTube

```typescript title="components/blog/mdx/youtube.tsx"
interface YouTubeProps {
  id: string;
  title?: string;
}

export function YouTube({ id, title = 'Video de YouTube' }: YouTubeProps) {
  return (
    <div className="relative aspect-video my-8 rounded-xl overflow-hidden">
      <iframe
        src={`https://www.youtube.com/embed/${id}`}
        title={title}
        allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture"
        allowFullScreen
        className="absolute inset-0 w-full h-full"
      />
    </div>
  );
}
```

### Registrar componentes MDX

```typescript title="components/blog/mdx-content.tsx"
import { MDXRemote } from 'next-mdx-remote/rsc';
import { Callout } from './mdx/callout';
import { YouTube } from './mdx/youtube';
import { Tweet } from './mdx/tweet';
import { CodeComparison } from './mdx/code-comparison';
import Image from 'next/image';
import Link from 'next/link';

const components = {
  // Componentes personalizados
  Callout,
  YouTube,
  Tweet,
  CodeComparison,

  // Sobrescribir elementos nativos
  img: (props: any) => (
    <Image
      {...props}
      width={800}
      height={400}
      className="rounded-xl my-8"
      alt={props.alt || ''}
    />
  ),
  a: (props: any) => {
    const isExternal = props.href?.startsWith('http');
    if (isExternal) {
      return (
        <a {...props} target="_blank" rel="noopener noreferrer">
          {props.children}
        </a>
      );
    }
    return <Link {...props}>{props.children}</Link>;
  },
};

interface MDXContentProps {
  content: string;
}

export function MDXContent({ content }: MDXContentProps) {
  return <MDXRemote source={content} components={components} />;
}
```

## Borradores

Para crear un borrador que no se publique, usa `published: false`:

```yaml
---
title: Artículo en progreso
published: false
---
```

Los borradores:
- No aparecen en la lista de posts
- No se incluyen en el sitemap
- No se indexan en Google
- Solo son accesibles con la URL directa en desarrollo

## Checklist antes de publicar

<Steps>
  <Step>
    ### Revisa el SEO
    - [ ] Título entre 50-60 caracteres
    - [ ] Descripción entre 150-160 caracteres
    - [ ] URL amigable (slug)
    - [ ] Imagen destacada con alt text
  </Step>

  <Step>
    ### Verifica el contenido
    - [ ] Sin errores ortográficos
    - [ ] Links funcionan correctamente
    - [ ] Imágenes cargan bien
    - [ ] Código tiene syntax highlighting
  </Step>

  <Step>
    ### Optimiza rendimiento
    - [ ] Imágenes comprimidas (WebP preferido)
    - [ ] Imágenes con dimensiones correctas
    - [ ] No hay componentes pesados innecesarios
  </Step>

  <Step>
    ### Publica
    - [ ] Cambiar `published: false` a `published: true`
    - [ ] Verificar fecha de publicación
    - [ ] Commit y deploy
  </Step>
</Steps>
