## Directorios principales

### `/app`

Contiene todas las rutas de la aplicación usando Next.js App Router.

| Directorio | Descripción |
|------------|-------------|
| `(auth)` | Páginas de autenticación (login, registro, etc.) |
| `(dashboard)` | Páginas protegidas del dashboard |
| `api` | API Routes para backend |

### `/components`

Componentes React reutilizables organizados por funcionalidad.

| Directorio | Descripción |
|------------|-------------|
| `ui` | Componentes base de shadcn/ui |
| `landing` | Componentes de la landing page |
| `auth` | Formularios y componentes de autenticación |

### `/lib`

Utilidades y configuraciones.

| Archivo | Descripción |
|---------|-------------|
| `utils.ts` | Función `cn()` para clases de Tailwind |
| `auth.ts` | Configuración de autenticación |
| `stripe.ts` | Cliente y utilidades de Stripe |
| `email.ts` | Configuración de Resend |

### `/prisma`

Esquema de base de datos y migraciones (solo para Better Auth + PostgreSQL).

## Archivos de configuración

| Archivo | Descripción |
|---------|-------------|
| `next.config.ts` | Configuración de Next.js |
| `tailwind.config.ts` | Configuración de Tailwind CSS |
| `components.json` | Configuración de shadcn/ui |
| `tsconfig.json` | Configuración de TypeScript |

## Convenciones de nombres

- **Componentes**: PascalCase (`LoginForm.tsx`)
- **Utilidades**: camelCase (`formatDate.ts`)
- **Rutas**: kebab-case (`/forgot-password`)
- **Archivos de página**: siempre `page.tsx`
- **Archivos de layout**: siempre `layout.tsx`
